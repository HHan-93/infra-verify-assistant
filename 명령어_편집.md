# 단일 명령어 프리셋

* OpenStack
  * 컴퓨트 (Nova)
    * 명령어 : openstack compute service list
      * 라벨 : 컴퓨트 서비스 상태
      * 설명 : Nova 데몬(compute, scheduler, conductor) 활성 상태 확인
    * 명령어 : openstack compute service set --disable --disable-reason "QA_Test" <HOST> nova-compute
      * 라벨 : 컴퓨트 노드 제어(차단)
      * 설명 : 특정 컴퓨트 노드 스케줄링 일시 차단 (유지보수/검증용)
    * 명령어 : openstack server list --all-projects --long
      * 라벨 : 인스턴스 목록(전체)
      * 설명 : 전체 프로젝트의 VM 상태 및 배치된 물리 호스트 노드 조회
    * 명령어 : openstack server list --all-projects --status ERROR
      * 라벨 : ERROR 인스턴스 검색
      * 설명 : 생성 실패하거나 오류 상태에 빠진 VM만 정확히 필터링
    * 명령어 : openstack server show <SERVER_ID>
      * 라벨 : 인스턴스 상세 정보
      * 설명 : VM의 생성 시간, 네트워크, 보안 그룹, 배치 호스트 상세 메타데이터
    * 명령어 : openstack console log show <SERVER_ID>
      * 라벨 : 인스턴스 콘솔 로그
      * 설명 : 커널 패닉, 부팅 에러 등 VM 초기화 콘솔 로그 확인
    * 명령어 : openstack server event list <SERVER_ID>
      * 라벨 : 인스턴스 액션 기록
      * 설명 : VM에 수행된 API 요청(생성/재부팅/마이그레이션) 이력 추적
    * 명령어 : openstack hypervisor list --long
      * 라벨 : 하이퍼바이저 리소스
      * 설명 : 각 물리 노드별 vCPU, RAM, 로컬 디스크 사용량 및 할당량
    * 명령어 : nova-manage cell_v2 discover_hosts
      * 라벨 : 셀(Cell) 호스트 갱신
      * 설명 : 신규 노드 증설 직후 컨트롤 플레인에 새 호스트를 즉시 인식시킴
    * 명령어 : openstack server migrate <SERVER_ID> --live-migration
      * 라벨 : 라이브 마이그레이션
      * 설명 : 다운타임 없이 VM을 다른 컴퓨트 노드로 실시간 이동
    * 명령어 : openstack server migration list
      * 라벨 : 마이그레이션 상태
      * 설명 : 마이그레이션 진행률 및 성공/실패 여부 확인
  * 네트워크 (Neutron·OVS)
    * 명령어 : openstack network agent list
      * 라벨 : 네트워크 에이전트
      * 설명 : L3/DHCP 에이전트의 alive 상태 및 호스트 매핑 확인
    * 명령어 : openstack network list --long
      * 라벨 : 네트워크 리스트 상세 조회
      * 설명 : 내부/외부 네트워크, 서브넷 매핑, MTU 및 Provider 정보
	* 명령어 : openstack subnet list
	  * 라벨 : 서브넷 목록 조회
	  * 설명 : 각 네트워크에 종속된 서브넷의 IP 대역(CIDR) 할당 현황 및 DHCP 활성화 여부 확인
    * 명령어 : openstack port show <PORT_ID>
      * 라벨 : 가상 포트 상세
      * 설명 : VM 가상 포트의 MAC, 바인딩 호스트, ACTIVE/DOWN 상태
    * 명령어 : openstack floating ip list
      * 라벨 : Floating IP 할당
      * 설명 : 외부 통신용 Public IP의 할당 및 연결 포트 상태 점검
    * 명령어 : openstack security group rule list
      * 라벨 : 보안 그룹 룰 상세
      * 설명 : Ingress/Egress 포트, 프로토콜, 타겟 IP 대역 룰 확인
    * 명령어 : ovs-vsctl show
      * 라벨 : OVS 브릿지 상태
      * 설명 : Open vSwitch의 br-int, br-ex, br-tun 브릿지 및 포트 매핑
    * 명령어 : ovs-ofctl dump-flows br-int
      * 라벨 : OVS 오픈플로우 룰
      * 설명 : VM 간 통신 및 보안 룰이 적용된 OVS OpenFlow 플로우 덤프
  * 스토리지 (Cinder)
    * 명령어 : openstack volume list --all-projects --long
      * 라벨 : 볼륨 목록(전체)
      * 설명 : 전체 볼륨 상태(in-use/available/error) 및 연결된 인스턴스
    * 명령어 : openstack volume show <VOLUME_ID>
      * 라벨 : 볼륨 상세 정보
      * 설명 : 볼륨 생성 실패 시 백엔드 스토리지 매핑 및 에러 코드 분석
    * 명령어 : openstack volume backup list
      * 라벨 : 볼륨 백업 리스트
      * 설명 : Cinder를 통해 생성된 볼륨 백업 이미지 및 상태 확인
	* 명령어 : openstack volume extend <VOLUME_ID> <NEW_SIZE>
	  * 라벨 : 볼륨 용량 확장
	  * 설명 : 기존에 생성된 Cinder 블록 스토리지 볼륨의 크기를 지정한 용량(GB)으로 온라인 확장
  * 이미지 (Glance)
    * 명령어 : openstack image list --long
      * 라벨 : 이미지 목록(전체)
      * 설명 : OS 이미지 상태, 포맷(qcow2/raw), 크기, 가시성 조회
	* 명령어 : openstack image create --file <FILE_PATH> --disk-format qcow2 --container-format bare --public <IMAGE_NAME>
	  * 라벨 : 신규 이미지 업로드
	  * 설명 : 로컬에 준비된 qcow2 등 OS 이미지 파일을 Glance 서비스에 신규 등록(public)
  * 메시지큐·DB
    * 명령어 : rabbitmqctl cluster_status
      * 라벨 : RabbitMQ 클러스터
      * 설명 : 메시지 큐 노드 상태 및 파티션(네트워크 단절) 발생 여부
    * 명령어 : rabbitmqctl list_queues | grep -v ' 0'
      * 라벨 : 메시지 큐 적체 확인
      * 설명 : 처리되지 못하고 큐에 쌓여있는 데드레터 메시지 목록 확인
    * 명령어 : mysql -u root -e "SHOW STATUS LIKE 'wsrep_cluster_size';"
      * 라벨 : MariaDB 갤러라 상태
      * 설명 : DB 이중화(Galera) 노드 개수 및 클러스터 싱크 정상 여부
  * 인증·쿼터 (Keystone)
    * 명령어 : openstack endpoint list
      * 라벨 : 엔드포인트 URL 점검
      * 설명 : 전체 서비스의 public, internal, admin API 엔드포인트 매핑
    * 명령어 : openstack project list
      * 라벨 : 프로젝트/테넌트 목록
      * 설명 : 클러스터 내 구성된 전체 프로젝트 및 도메인 확인
    * 명령어 : openstack role assignment list --names
      * 라벨 : 유저 권한 매핑
      * 설명 : 특정 유저가 어느 프로젝트에서 admin/member 권한을 가지는지 조회
    * 명령어 : openstack quota show --default
      * 라벨 : 테넌트 리소스 쿼터
      * 설명 : 프로젝트별 최대 생성 가능한 VM, 볼륨, IP 제한량 조회
  * 하이퍼바이저 (Libvirt)
    * 명령어 : virsh list --all
      * 라벨 : 호스트 KVM VM 상태
      * 설명 : 컴퓨트 노드 쉘에서 하이퍼바이저 레벨의 실제 VM 구동 상태 조회
	* 명령어 : virsh dominfo <INSTANCE_ALIAS>
	  * 라벨 : 인스턴스 상태
	  * 설명 : 인스턴스의 CPU, 메모리, 상태 등을 조회
    * 명령어 : virsh domblklist <INSTANCE_NAME>
      * 라벨 : VM 디스크 매핑 확인
      * 설명 : VM에 매핑된 실제 블록 디바이스(RBD 경로 등) 식별
	* 명령어 : virsh dumpxml <INSTANCE_ALIAS>
	  * 라벨 : xml 설정 파일 직접 확인
	  * 설명 : 인스턴스에 적용된 모든 설정(네트워크, 디스크, CPU, 메모리 등)을 원본 XML 형태로 조회
  * 로그
    * 명령어 : tail -n 100 /var/log/nova/nova-api.log | grep -i error
      * 라벨 : Nova 에러 로그
      * 설명 : VM 생성 실패 시 가장 먼저 확인할 Nova API 최근 에러 로그
    * 명령어 : tail -n 100 /var/log/neutron/neutron-server.log | grep -i error
      * 라벨 : Neutron 에러 로그
      * 설명 : 네트워크/포트 바인딩 실패 시 원인 파악용 에러 로그
    * 명령어 : tail -n 100 /var/log/cinder/cinder-volume.log | grep -i error
      * 라벨 : Cinder 에러 로그
      * 설명 : 볼륨 생성 및 연결 실패 시 스토리지 연동 에러 로그

* Ceph
  * 클러스터
    * 명령어 : ceph -s
      * 라벨 : 클러스터 요약(s)
      * 설명 : HEALTH, MON/OSD, PG, 용량 상태를 확인하는 최우선 점검 명령
    * 명령어 : ceph health detail
      * 라벨 : 클러스터 헬스 상세
      * 설명 : WARN/ERR 발생 시 크래시 데몬이나 손상된 PG 등 근본 원인 출력
    * 명령어 : ceph -w
      * 라벨 : 실시간 이벤트(w)
      * 설명 : PG 변경, OSD Up/Down, 리밸런싱 등 실시간 모니터링 (종료: Ctrl+C)
    * 명령어 : ceph quorum_status --format json-pretty
      * 라벨 : MON 쿼럼 상태
      * 설명 : 모니터 노드 선출 상태 및 쿼럼 정상 형성 딥다이브 (JSON)
    * 명령어 : ceph mgr module ls
      * 라벨 : MGR 모듈 상태
      * 설명 : Dashboard, Prometheus 익스포터 등 활성화된 MGR 플러그인 상태
    * 명령어 : ceph orch ps
      * 라벨 : 데몬 컨테이너 목록
      * 설명 : Cephadm 오케스트레이터로 배포된 OSD/MON/MGR 데몬 컨테이너 상태
    * 명령어 : tail -n 200 /var/log/ceph/ceph.log | grep -iE 'err|warn'
      * 라벨 : 클러스터 에러 로그
      * 설명 : 최근 200줄 로그에서 클러스터 에러와 경고 알람만 필터링
  * OSD
    * 명령어 : ceph osd tree
      * 라벨 : OSD 트리/가중치
      * 설명 : CRUSH 맵 기반 호스트 노드별 OSD 트리와 weight, up/down 상태
    * 명령어 : ceph osd tree -f json-pretty
      * 라벨 : OSD 트리 JSON
      * 설명 : OSD 구성도를 JSON으로 출력하여 파싱/자동화 스크립트 연동
    * 명령어 : ceph osd df
      * 라벨 : 디스크 사용률(OSD)
      * 설명 : OSD별 물리 디스크 사용량, 여유 공간, 데이터 분산 편차(variance)
	* 명령어 : ceph osd pool autoscale-status
	  * 라벨 : PG 오토스케일 상태
	  * 설명 : 각 풀별 데이터 타겟 사이즈 대비 현재 PG 개수의 적절성 및 자동 확장 데몬의 동작 상태 확인
    * 명령어 : ceph osd perf
      * 라벨 : OSD 데몬 성능 지연
      * 설명 : 각 OSD의 commit/apply 지연 시간(Latency) 추적 (느린 디스크 색출)
    * 명령어 : ceph osd set noout
      * 라벨 : 자동 아웃(Out) 방지
      * 설명 : 노드 재부팅/점검 전 OSD가 Out되어 불필요한 리밸런싱이 발생하지 않도록 홀드
    * 명령어 : ceph osd unset noout
      * 라벨 : 자동 아웃 방지 해제
      * 설명 : 점검 완료 후 noout 플래그 해제하여 정상 관리 상태 복구
    * 명령어 : ceph osd scrub <OSD_ID>
      * 라벨 : 강제 딥 스크럽
      * 설명 : 특정 OSD 데이터 정합성 검사(Scrub) 강제 수행 예약
    * 명령어 : ceph crash ls
      * 라벨 : OSD 크래시 기록
      * 설명 : 비정상 종료되거나 크래시된 OSD 데몬 이력 확인
    * 명령어 : ceph crash archive-all
      * 라벨 : 크래시 알람 초기화
      * 설명 : 확인이 끝난 크래시 로그를 보관 처리하여 클러스터 경고 알람 해제
  * PG
    * 명령어 : ceph pg stat
      * 라벨 : PG 통계 요약
      * 설명 : PG 맵 버전 및 클라이언트 Read/Write IOPS, Throughput 요약
    * 명령어 : ceph pg dump_stuck inactive
      * 라벨 : 비정상(Stuck) PG
      * 설명 : 데이터 I/O 처리가 멈춰있는(stuck) Placement Group 확인
    * 명령어 : ceph pg dump_stuck degraded
      * 라벨 : 복구중(Degraded) PG
      * 설명 : 노드/OSD 장애로 데이터 리플리케이션이 진행 중인 PG 확인
    * 명령어 : ceph pg repair <PG_ID>
      * 라벨 : 손상된 PG 수동 복구
      * 설명 : Inconsistent 에러가 뜬 특정 PG에 대해 수동 복구 커맨드 전송
  * 풀/RADOS
    * 명령어 : ceph osd pool ls detail
      * 라벨 : 풀 리스트 상세
      * 설명 : 전체 풀의 size, min_size, crush rule, pg_num 등 세부 설정값
    * 명령어 : ceph df detail
      * 라벨 : 풀별 용량(ceph df)
      * 설명 : 풀별 사용량, 가용량 및 저장된 오브젝트 수 요약
    * 명령어 : rados df
      * 라벨 : 논리적 데이터 용량
      * 설명 : 풀에 저장된 객체 수, 크기 및 논리/물리적 데이터 사용률 분석
    * 명령어 : ceph osd pool stats
      * 라벨 : 풀 실시간 I/O 부하
      * 설명 : 각 풀에서 발생하는 실시간 Client Read/Write I/O 모니터링
    * 명령어 : rados bench -p <POOL_NAME> 60 write --no-cleanup
      * 라벨 : RADOS 성능 벤치마크
      * 설명 : 특정 풀 대상 60초 쓰기 부하 테스트로 Throughput/IOPS 성능 한계 검증
    * 명령어 : rados -p <POOL_NAME> ls
      * 라벨 : 저장된 오브젝트 리스트
      * 설명 : 특정 풀에 저장된 오브젝트 청크 리스트 실제 조회 (데이터 유실 검증)
    * 명령어 : ceph osd map <POOL_NAME> <OBJECT>
      * 라벨 : 오브젝트 OSD 추적
      * 설명 : 특정 파일(오브젝트)이 실제 어느 OSD에 나뉘어 저장되는지 매핑 추적
  * 블록 (RBD)
    * 명령어 : rbd ls -p <POOL_NAME>
      * 라벨 : RBD 이미지 리스트
      * 설명 : 풀에 생성된 블록 디바이스(VM 디스크 등) 이미지 조회
    * 명령어 : rbd info <IMAGE_NAME> -p <POOL_NAME>
      * 라벨 : RBD 이미지 상세
      * 설명 : RBD 이미지 실제 크기, 객체 크기(기본 4M), 포맷, Lock 상태
	* 명령어 : rbd snap create <POOL_NAME>/<IMAGE_NAME>@<SNAP_NAME>
	  * 라벨 : RBD 스냅샷 생성
	  * 설명 : 특정 블록 디바이스의 현재 상태를 스냅샷으로 즉각 보존하여 데이터 백업 및 롤백 지점 확보
    * 명령어 : rbd trash ls -p <POOL_NAME>
      * 라벨 : 삭제 대기 중인 RBD
      * 설명 : 삭제 명령 수신 후 백그라운드 정리 대기 중인 이미지 목록

* Kubernetes
  * 노드 (Node)
    * 명령어 : kubectl get nodes -o wide
      * 라벨 : 노드 상태 및 IP
      * 설명 : 워커/마스터 노드의 Ready 상태, 내부 IP, 커널 및 런타임 버전
    * 명령어 : kubectl describe nodes
      * 라벨 : 노드 상세 리소스
      * 설명 : 노드 Taint, 스케줄링된 파드, CPU/Mem 요청(Request) 및 제한량
    * 명령어 : kubectl get nodes -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.spec.taints}{"\n"}{end}'
      * 라벨 : 노드 테인트(Taint) 확인
      * 설명 : 파드 스케줄링을 막는 오염(Taint) 설정값만 추출하여 원인 분석
    * 명령어 : kubectl drain <NODE_NAME> --ignore-daemonsets --delete-emptydir-data
      * 라벨 : 노드 안전 퇴출(Drain)
      * 설명 : 유지보수 전 실행 중인 파드를 다른 노드로 안전하게 대피(Eviction)
    * 명령어 : kubectl cordon <NODE_NAME>
      * 라벨 : 스케줄링 일시 차단
      * 설명 : 파드 대피 없이 해당 노드에 신규 파드가 생성되는 것만 막음
  * 파드 (Pod)
    * 명령어 : kubectl get pods -A -o wide
      * 라벨 : 전체 파드 목록
      * 설명 : 모든 네임스페이스 파드의 상태, 배치된 노드, 할당 IP 조회
    * 명령어 : kubectl get pods -A --field-selector=status.phase!=Running
      * 라벨 : 비정상 파드 필터링
      * 설명 : Pending, Error, Crash 상태인 결함 파드만 정확하게 필터링
    * 명령어 : kubectl describe pod <POD_NAME> -n <NAMESPACE>
      * 라벨 : 파드 상세 분석
      * 설명 : ImagePullBackOff, 스케줄링 실패 사유 등 파드 상세 이벤트 로그
	* 명령어 : kubectl get pods -w
	  * 라벨 : 파드 상태 실시간 감시(Watch)
	  * 설명 : 파드의 생성, 종료, 에러(Crash) 등 상태 변화 이벤트를 실시간 스트림으로 모니터링
    * 명령어 : kubectl get pods -l app=<APP_NAME> -n <NAMESPACE>
      * 라벨 : 라벨 기준 파드 조회
      * 설명 : 특정 라벨(예: 앱 이름)을 가진 파드 그룹만 모아서 조회
    * 명령어 : kubectl top nodes
      * 라벨 : 노드 리소스 사용량
      * 설명 : 노드의 현재 CPU/메모리 실제 사용량(Metrics Server 기반)
    * 명령어 : kubectl top pods -A
      * 라벨 : 파드 리소스 사용량
      * 설명 : 파드의 현재 CPU/메모리 실제 사용량(Metrics Server 기반)
    * 명령어 : kubectl logs <POD_NAME> -n <NAMESPACE> --previous
      * 라벨 : 크래시 이전 로그
      * 설명 : 컨테이너가 죽었을 때 재시작 직전 컨테이너가 남긴 에러 로그 추출
    * 명령어 : kubectl exec -it <POD_NAME> -n <NAMESPACE> -- /bin/sh
      * 라벨 : 파드 내부 쉘 접속
      * 설명 : 파드 컨테이너 내부에 직접 접근하여 설정/네트워크 상태 검증
    * 명령어 : kubectl cp <NAMESPACE>/<POD_NAME>:<PATH> <LOCAL_PATH>
      * 라벨 : 파드 내부 파일 복사
      * 설명 : 컨테이너 안의 힙 덤프, 로그 등 디버깅 파일을 로컬로 복사
    * 명령어 : kubectl get all -n <NAMESPACE>
      * 라벨 : 전체 리소스 일괄 조회
      * 설명 : 특정 네임스페이스의 Pod, Service, Deployment, ReplicaSet 일괄
  * 네트워크
    * 명령어 : kubectl get endpoints -A
      * 라벨 : 서비스 엔드포인트
      * 설명 : Service와 실제 Pod IP 매핑이 정상적으로 로드밸런싱 되는지 검증
    * 명령어 : kubectl get ingress -A
      * 라벨 : Ingress 라우팅 룰
      * 설명 : 외부 트래픽을 내부 서비스로 연결하는 도메인 및 Path 라우팅 룰
    * 명령어 : kubectl run -i --tty --rm debug --image=busybox --restart=Never -- nslookup kubernetes.default
      * 라벨 : DNS 질의 테스트
      * 설명 : 임시 파드를 생성해 CoreDNS의 클러스터 내부 네임 레졸루션 점검
    * 명령어 : kubectl port-forward svc/<SVC_NAME> <LOCAL_PORT>:<SVC_PORT>
      * 라벨 : 로컬 포트 포워딩
      * 설명 : 외부 노출이 없는 백엔드 서비스 UI/API를 로컬 PC 포트로 임시 연결
  * 스토리지
    * 명령어 : kubectl get pv,pvc -A

      * 설명 : 프로비저닝된 물리 볼륨(PV)과 파드의 볼륨 클레임(PVC) Bound 상태
    * 명령어 : kubectl get sc
      * 라벨 : 스토리지 클래스

      * 라벨 : 디플로이먼트 상태

    * 명령어 : kubectl rollout undo deployment/<NAME> -n <NAMESPACE>
      * 라벨 : 디플로이먼트 롤백
      * 설명 : 신규 배포 결함 시 직전의 정상 리비전 파드로 즉시 롤백
    * 명령어 : kubectl scale deployment <NAME> --replicas=<NUM> -n <NAMESPACE>

      * 설명 : 수동으로 파드 개수를 늘리거나 줄여 부하 대응 및 동작 검증
    * 명령어 : kubectl rollout restart deployment/<NAME> -n <NAMESPACE>
      * 라벨 : 재배포(Restart)
      * 설명 : 기존 파드를 하나씩 재생성하여 ConfigMap 등 변경 설정 갱신
    * 명령어 : helm list -A
      * 라벨 : Helm 릴리스 목록
      * 설명 : Helm 차트로 배포된 서비스의 버전, 상태(deployed/failed) 확인
    * 명령어 : helm get values <RELEASE_NAME> -n <NAMESPACE>
      * 라벨 : Helm 주입 변수 덤프
      * 설명 : 차트 배포 시 사용자가 주입한 values.yaml 커스텀 설정값 전체 출력

    * 명령어 : kubectl get events -A --sort-by='.lastTimestamp'

* 시스템·부하검증
  * 시스템/프로세스

      * 라벨 : 상위 프로세스(top)
      * 설명 : 시스템 자원을 과다 점유하는 상위 20개 프로세스 스냅샷
	* 명령어 : free -h
	  * 라벨 : 시스템 메모리 사용량
	  * 설명 : 전체 RAM 및 Swap 메모리의 총량, 사용량, 그리고 리눅스 캐시(buff/cache) 여유분 조회
    * 명령어 : dstat -cdngy 1
      * 라벨 : 통합 실시간 뷰(dstat)
      * 설명 : 1초 단위로 CPU, 디스크 I/O, 네트워크 대역폭, 페이징 종합 출력 (Ctrl+C)
    * 명령어 : dmesg -T --level=err,crit,alert,emerg
      * 라벨 : 커널 패닉/OOM(dmesg)
      * 설명 : OOM Killer, 디스크 배드섹터 등 치명적인 커널 로그만 추출
    * 명령어 : iostat -xz 1 5
      * 라벨 : 디스크 I/O 병목(iostat)
      * 설명 : 블록 디바이스별 Read/Write 속도와 await(지연) 5회 측정
    * 명령어 : sar -u 1 5
      * 라벨 : CPU 누적 부하(sar)

    * 명령어 : sar -n DEV 1 5
      * 라벨 : 네트워크 부하(sar)
      * 설명 : sysstat 기반 인터페이스별 네트워크 사용량 5회 측정
    * 명령어 : vmstat 1 5
      * 라벨 : 페이징/IO Wait(vmstat)
      * 설명 : 메모리 스왑(si/so) 상태와 디스크 지연으로 인한 대기(wa) 추적
    * 명령어 : lsof +D /var/log
      * 라벨 : 점유 디렉토리 추적(lsof)
      * 설명 : 특정 디렉토리를 점유하여 삭제를 막는 프로세스 식별
    * 명령어 : lsof -i :80
      * 라벨 : 포트 점유 추적(lsof)
      * 설명 : 특정 포트를 점유 중인 프로세스 식별
    * 명령어 : journalctl -p 3 -xb
      * 라벨 : 부팅 세션 에러 로그
      * 설명 : 현재 부팅 세션에서 발생한 Error 등급(-p 3) 이상의 시스템 로그
    * 명령어 : sysctl -a | grep -i 'net.ipv4\|vm.swappiness'
      * 라벨 : 런타임 커널 변수
      * 설명 : 메모리 스왑 빈도 및 TCP 네트워크 튜닝 설정 적용 여부
    * 명령어 : sync; echo 3 > /proc/sys/vm/drop_caches

      * 설명 : 임계치 테스트를 위해 PageCache/inode 캐시를 비워 가용 메모리 확보
  * 부하 테스트
    * 명령어 : stress-ng --cpu 8 --vm 4 --vm-bytes 2G --timeout 120s
      * 라벨 : CPU/메모리 부하 유도
      * 설명 : 알람 연동 테스트를 위한 2분간 CPU 점유 및 8GB 램 할당 부하

    * 명령어 : iperf3 -s -D
      * 라벨 : 대역폭 테스트(서버)
      * 설명 : 랙 간 대역폭/지연 측정을 위한 iperf3 백그라운드 리슨 대기
    * 명령어 : iperf3 -c <SERVER_IP> -t 60 -P 4
      * 라벨 : 대역폭 테스트(클라)
      * 설명 : 60초간 4개 병렬 스레드로 타겟 노드와의 TCP 최대 실효 대역폭 측정
  * 네트워크
    * 명령어 : tcpdump -i any port 80 -n -c 100
      * 라벨 : 패킷 캡처(tcpdump)
      * 설명 : 특정 포트로 인입되는 패킷 헤더와 출발지 IP 100개 캡처
    * 명령어 : ss -tunlpo
      * 라벨 : TCP 소켓/포트(ss)
      * 설명 : 서버의 TCP/UDP 리슨 포트, 바인딩 프로세스, 타이머 상태 출력
    * 명령어 : netstat -nat | grep TIME_WAIT | wc -l
      * 라벨 : TIME_WAIT 소켓 점검
      * 설명 : 연결 종료 후 남은 잔여 소켓 수를 카운트하여 커널 튜닝 판단
    * 명령어 : ip route show
      * 라벨 : 라우팅 테이블
      * 설명 : 디폴트 게이트웨이 및 대역별 패킷 우회 라우팅 인터페이스 검증
    * 명령어 : cat /proc/net/bonding/bond0

      * 라벨 : 랜카드 하드웨어 스펙
      * 설명 : 물리 랜카드 지원 속도(10G/40G), Duplex 상태 및 Link up 확인
    * 명령어 : mtr <TARGET_IP>
      * 라벨 : 실시간 라우팅 지연(mtr)
      * 설명 : Ping+Traceroute 결합으로 목적지까지 구간별 패킷 손실률 추적
    * 명령어 : dig @<DNS_IP> <DOMAIN> +short
      * 라벨 : DNS 도메인 쿼리(dig)
      * 설명 : 특정 네임서버를 지정하여 도메인의 A/CNAME 응답 직접 검증
  * 디스크·하드웨어
    * 명령어 : lsblk -f
      * 라벨 : 물리 디스크 파티션
      * 설명 : 노드 장착 디스크 블록 구조, UUID, 파일시스템 타입(XFS/EXT4) 매핑
    * 명령어 : df -iTh
      * 라벨 : 디스크/Inode 사용률
      * 설명 : 디스크 여유 공간 + 작은 파일 과다로 인한 Inode 고갈 여부 점검
    * 명령어 : du -sh /var/* | sort -rh | head -10
      * 라벨 : 대용량 디렉토리 색출
      * 설명 : 특정 파티션에서 용량을 가장 많이 차지하는 하위 폴더 10개 추출
    * 명령어 : lscpu
      * 라벨 : CPU/NUMA 스펙
      * 설명 : 물리 코어, 쓰레드, NUMA 노드 매핑 및 가상화(VT-x) 지원 유무
    * 명령어 : dmidecode -t memory | grep -i 'size\|speed'
      * 라벨 : 물리 메모리(RAM) 스펙
      * 설명 : 메인보드 뱅크에 장착된 램 모듈의 개별 크기와 동작 클럭
    * 명령어 : iptables -L -n -v | head -n 30
      * 라벨 : 방화벽(Iptables) 룰
      * 설명 : 커널 레벨 패킷 필터링 룰의 Drop/Accept 카운트 확인

* 리눅스 코어
  * 이동·파일 기본
    * 명령어 : pwd
      * 라벨 : 현재 위치 확인
      * 설명 : 지금 내가 있는 디렉토리의 전체 경로 출력
    * 명령어 : cd <경로>
      * 라벨 : 폴더 이동
      * 설명 : 지정 폴더로 이동 (.. 상위, ~ 홈)
    * 명령어 : ls -alh
      * 라벨 : 파일/폴더 목록 상세
      * 설명 : 숨김 파일 포함 권한, 소유자, 크기를 읽기 쉬운 단위로 출력
    * 명령어 : mkdir -p <폴더명>
      * 라벨 : 폴더 생성
      * 설명 : 중간 경로(상위 폴더)까지 한 번에 새 폴더 생성
    * 명령어 : find /var/log -name "*.log" -type f
      * 라벨 : 하위 파일 전체 검색
      * 설명 : 경로 아래 .log 확장자 파일을 재귀적으로 검색
    * 명령어 : find /tmp -name "*.tmp" -mtime +7 -exec rm -f {} \;
      * 라벨 : 검색 후 일괄 삭제
      * 설명 : 7일 이상 지난 .tmp 찌꺼기 파일을 찾아 일괄 삭제
    * 명령어 : cp -r <원본> <대상>
      * 라벨 : 파일/폴더 복사
      * 설명 : 파일이나 폴더(-r)를 다른 위치로 복사
    * 명령어 : mv <원본> <대상>
      * 라벨 : 이동/이름 변경
      * 설명 : 파일·폴더를 옮기거나 이름 변경 (같은 명령으로 둘 다)
    * 명령어 : rm -ri <대상>
      * 라벨 : 삭제(주의)
      * 설명 : 확인하며(-i) 폴더 포함(-r) 영구 삭제 — 되돌릴 수 없으니 주의
	* 명령어 : ln -s <원본_경로> <링크_이름>
	  * 라벨 : 심볼릭 링크 생성
	  * 설명 : 복잡한 경로나 파일에 대한 바로가기(소프트 링크)를 생성하여 쉘 접근성을 단순화
  * 내용·텍스트 처리
    * 명령어 : cat <파일명>
      * 라벨 : 파일 내용 출력
      * 설명 : 짧은 텍스트 파일 내용 전체를 화면에 출력
	* 명령어 : watch -n 1 <명령어>
	  * 라벨 : 명령어 주기적 반복 실행
	  * 설명 : 특정 명령어(예: ls, df, kubectl get 등)를 1초 주기로 반복 실행하여 결과 변화를 화면에 실시간 갱신
    * 명령어 : less <파일명>
      * 라벨 : 대용량 파일 페이징
      * 설명 : 대용량 로그를 멈춤 없이 위/아래로 스크롤 열람 (q 종료)
    * 명령어 : tail -f <파일명>
      * 라벨 : 로그 실시간 모니터링
      * 설명 : 실시간으로 추가되는 로그를 꼬리 물며 확인 (Ctrl+C 종료)
    * 명령어 : grep -i "error" <파일명>
      * 라벨 : 단어 검색(grep)
      * 설명 : 대소문자 무시(-i)하고 "error"가 포함된 모든 줄 필터링
    * 명령어 : awk '{print $1, $9}' access.log
      * 라벨 : 열 파싱(awk)
      * 설명 : 공백 기준으로 로그를 쪼개 1번째(IP)·9번째(상태코드) 열만 추출
    * 명령어 : sed -n '100,200p' server.log
      * 라벨 : 라인 구간 출력(sed)
      * 설명 : 수십만 줄 중 정확히 100~200번 줄 사이만 잘라내기
    * 명령어 : cat list.txt | xargs -I {} rm -rf {}
      * 라벨 : 파이프 반복(xargs)
      * 설명 : 앞 명령의 목록 결과를 하나씩 뒤 명령으로 넘겨 대량 작업
  * 압축·전송
    * 명령어 : tar -czvf backup.tar.gz /etc/
      * 라벨 : 폴더 압축(tar.gz)
      * 설명 : 폴더를 하나의 파일로 묶고 gzip으로 용량 압축
    * 명령어 : tar -xzvf backup.tar.gz -C /tmp/
      * 라벨 : 압축 해제
      * 설명 : tar.gz를 지정 경로(/tmp/)에 원본 구조 그대로 해제
    * 명령어 : curl -O <URL>
      * 라벨 : 웹 파일 다운로드
      * 설명 : 외부/내부 레포에서 스크립트·바이너리 다운로드 (또는 wget <URL>)
    * 명령어 : scp <파일명> user@<IP>:<경로>
      * 라벨 : 원격 안전 복사(scp)
      * 설명 : SSH로 다른 서버에 설정 파일 등을 안전하게 전송
    * 명령어 : rsync -avz /data/ /backup/
      * 라벨 : 디렉토리 동기화(rsync)
      * 설명 : 변경된 파일만 식별해 증분 백업 — 복제 시간 단축
  * 권한·프로세스
    * 명령어 : chmod 755 <스크립트.sh>
      * 라벨 : 권한(퍼미션) 변경
      * 설명 : 실행(x) 권한을 부여하여 쉘에서 구동 가능하게 변경
    * 명령어 : chown -R user:group <폴더명>
      * 라벨 : 소유권 갱신
      * 설명 : 폴더와 하위 전체의 소유자/그룹을 일괄 변경
    * 명령어 : nohup <명령어> &
      * 라벨 : 백그라운드 실행
      * 설명 : 터미널을 닫아도 백그라운드 스크립트가 죽지 않고 유지
    * 명령어 : ps aux | grep <데몬명>
      * 라벨 : 프로세스 PID 검색
      * 설명 : 실행 중인 서비스의 유저, CPU/메모리율 및 PID 검색
    * 명령어 : kill -9 <PID>
      * 라벨 : 프로세스 강제 종료
      * 설명 : 응답이 멈춘(좀비) 프로세스를 PID 기반으로 강제 Kill
  * 서비스·패키지
    * 명령어 : systemctl status <서비스명>
      * 라벨 : 서비스 상태
      * 설명 : systemd 데몬(sshd, nginx 등)의 활성/에러/중지 상태 상세
    * 명령어 : systemctl enable --now <서비스명>
      * 라벨 : 서비스 자동 시작 등록
      * 설명 : 즉시 시작 + 재부팅 시 자동으로 올라오게 등록
    * 명령어 : apt update
      * 라벨 : 패키지 갱신(Ubuntu)
      * 설명 : 레포지토리에서 설치 가능한 최신 패키지 정보 갱신
    * 명령어 : dnf check-update
      * 라벨 : 패키지 갱신(RHEL)
      * 설명 : RHEL 계열 최신 패키지 버전/의존성 갱신
    * 명령어 : apt install -y <패키지>
      * 라벨 : 프로그램 설치
      * 설명 : 터미널 툴(htop, vim 등) 설치 (-y로 묻지 않고 진행)
    * 명령어 : apt remove <패키지>
      * 라벨 : 프로그램 제거
      * 설명 : 불필요한 패키지 제거
    * 명령어 : dpkg -l | grep <이름>
      * 라벨 : 설치 검색(Ubuntu)
      * 설명 : 해당 라이브러리/툴이 설치되어 있는지 로컬 DB 검색
    * 명령어 : rpm -qa | grep <이름>
      * 라벨 : 설치 검색(RHEL)
      * 설명 : RHEL 계열에서 설치된 패키지 검색
  * 도움말·기록
    * 명령어 : man <명령어>
      * 라벨 : 명령어 매뉴얼
      * 설명 : 명령어의 옵션 플래그와 사용 예제 공식 매뉴얼 (q 종료)
    * 명령어 : which <명령어>
      * 라벨 : 명령어 위치 추적
      * 설명 : PATH에 등록된 실행 파일의 실제 바이너리 절대 경로 확인
    * 명령어 : history | grep <키워드>
      * 라벨 : 이전 명령어 기록
      * 설명 : 과거 입력했던 긴 명령어를 다시 찾아서 재사용


# 시나리오 흐름 명령어

* 공통 · 스토리지
  * 디스크 마운트 및 fstab 영구 등록
    * 요약 : 새로 추가한 디스크를 포맷·마운트하고, /etc/fstab 에 등록해 재부팅 후에도 마운트가 유지되도록 합니다.
    * 1. 현재 디스크/파티션 확인
      * 명령어 : lsblk -f
      * 설명 : 연결된 디스크와 파일시스템을 확인합니다. 새로 추가한 디스크(예: /dev/vdb)가 마운트/포맷 안 된 상태로 보이는지 확인하세요.
    * 2. 파일시스템 생성
      * 명령어 : sudo mkfs.ext4 /dev/<DISK>
      * 설명 : 새 디스크에 ext4 파일시스템을 만듭니다. <DISK> 는 1단계에서 확인한 장치명(vdb 등)으로 바꾸세요.
      * 주의 : ⚠️ 해당 디스크의 기존 데이터가 모두 삭제됩니다. 이미 포맷된 디스크라면 이 단계는 건너뜁니다.
    * 3. 마운트할 폴더 생성
      * 명령어 : sudo mkdir -p /mnt/data
      * 설명 : 디스크를 연결할 디렉토리를 만듭니다. -p 옵션으로 상위 경로까지 한 번에 생성됩니다.
    * 4. 디스크 마운트
      * 명령어 : sudo mount /dev/<DISK> /mnt/data
      * 설명 : 디스크를 위에서 만든 폴더에 임시로 마운트합니다.
    * 5. 마운트 확인
      * 명령어 : df -h /mnt/data
      * 설명 : 용량이 표시되면 정상적으로 마운트된 것입니다. `lsblk -f` 로도 마운트 지점을 확인할 수 있습니다.
    * 6. UUID 확인
      * 명령어 : sudo blkid /dev/<DISK>
      * 설명 : fstab 등록에 사용할 디스크 고유 UUID 를 확인합니다.
    * 7. fstab 등록
      * 명령어 : echo 'UUID=<UUID>  /mnt/data  ext4  defaults  0  2' | sudo tee -a /etc/fstab
      * 설명 : 재부팅 후에도 자동 마운트되도록 /etc/fstab 에 등록합니다. <UUID> 를 6단계에서 확인한 값으로 바꾸세요.
      * 주의 : 상단 [설정파일] 버튼으로 /etc/fstab 을 직접 열어 편집할 수도 있습니다.
    * 8. fstab 검증 및 적용
      * 명령어 : sudo mount -a
      * 설명 : fstab 의 문법 오류를 검사하고 전체 항목을 다시 마운트합니다. 오류 메시지가 없으면 정상입니다.
      * 주의 : ⚠️ 오류가 나면 fstab 항목이 잘못된 것입니다 — 재부팅 전에 반드시 수정하세요(부팅 실패 방지).
  * 마운트 해제 및 fstab 정리
    * 요약 : 디스크 마운트를 해제하고, fstab 등록을 제거해 영구적으로 분리합니다.
    * 1. 사용 중인 프로세스 확인
      * 명령어 : sudo lsof /mnt/data
      * 설명 : 해당 경로를 사용 중인 프로세스가 있으면 마운트 해제가 실패합니다. 결과가 있으면 종료 후 진행하세요.
    * 2. 마운트 해제
      * 명령어 : sudo umount /mnt/data
      * 설명 : 디스크 마운트를 해제합니다.
      * 주의 : "target is busy" 오류 시 1단계로 돌아가 점유 프로세스를 정리하세요.
    * 3. 해제 확인
      * 명령어 : lsblk -f
      * 설명 : /mnt/data 마운트 지점이 사라졌는지 확인합니다.
    * 4. fstab 등록 제거
      * 명령어 : sudo vi /etc/fstab
      * 설명 : 영구적으로 분리하려면 fstab 에서 해당 디스크 줄을 삭제합니다. 안 지우면 재부팅 시 다시 마운트를 시도합니다.
      * 주의 : 상단 [설정파일] 버튼으로 편집하는 것이 더 편하고 안전합니다.
  * LVM 구성 및 무중단 용량 확장
    * 요약 : PV/VG/LV 를 생성해 마운트한 뒤, 디스크를 추가해 무중단으로 용량을 확장하고 파일시스템을 온라인 리사이즈합니다.
    * 1. 도구 설치 (필요 시)
      * 명령어 : sudo apt install -y lvm2
      * 설명 : LVM 관리 도구를 설치합니다(대개 기본 설치되어 있음).
      * 주의 : RHEL/CentOS 계열은 `sudo dnf install -y lvm2`
    * 2. 디스크 인식 확인
      * 명령어 : lsblk
      * 설명 : LVM 으로 구성할 새 디스크(vdc 등)가 인식됐는지 확인합니다.
    * 3. 물리 볼륨(PV) 생성
      * 명령어 : sudo pvcreate /dev/<DISK>
      * 설명 : 디스크를 LVM 물리 볼륨으로 초기화합니다.
      * 주의 : ⚠️ 해당 디스크의 기존 데이터가 삭제됩니다.
    * 4. 볼륨 그룹(VG) 생성
      * 명령어 : sudo vgcreate data_vg /dev/<DISK>
      * 설명 : PV 들을 묶는 볼륨 그룹 data_vg 를 만듭니다.
    * 5. 논리 볼륨(LV) 생성
      * 명령어 : sudo lvcreate -l 100%FREE -n data_lv data_vg
      * 설명 : VG 의 남은 공간 전부로 논리 볼륨 data_lv 를 만듭니다.
    * 6. 파일시스템 생성
      * 명령어 : sudo mkfs.xfs /dev/data_vg/data_lv
      * 설명 : LV 에 xfs 파일시스템을 만듭니다. (ext4 를 쓰려면 mkfs.ext4)
    * 7. 마운트
      * 명령어 : sudo mkdir -p /mnt/data && sudo mount /dev/data_vg/data_lv /mnt/data
      * 설명 : 마운트 디렉토리를 만들고 LV 를 마운트합니다.
    * 8. (확장) VG 에 디스크 추가
      * 명령어 : sudo vgextend data_vg /dev/<NEW_DISK>
      * 설명 : 용량이 부족해지면 새 디스크를 PV 로 만든 뒤 VG 에 추가합니다. (먼저 pvcreate 필요)
    * 9. (확장) LV 용량 확장
      * 명령어 : sudo lvextend -l +100%FREE /dev/data_vg/data_lv
      * 설명 : VG 의 늘어난 공간만큼 LV 를 확장합니다(무중단).
    * 10. (확장) 파일시스템 온라인 리사이즈
      * 명령어 : sudo xfs_growfs /mnt/data
      * 설명 : 마운트된 상태에서 파일시스템을 확장합니다. xfs 는 xfs_growfs, ext4 는 resize2fs 사용.

    * 7. 복구 진행 감시
      * 명령어 : watch -n 5 'ceph -s'
      * 설명 : OSD 가 up 으로 전환되고 복구(recovery/backfill)가 진행·완료되는지 실시간 감시합니다. (종료: Ctrl+C)

      * 설명 : 원인을 수정(이미지/리소스/설정)한 뒤 무중단으로 파드를 재생성합니다.

    * 6. CPU/메모리 부하 발생 (수치는 환경에 맞게 조절)
      * 명령어 : stress-ng --cpu 4 --vm 2 --vm-bytes 1G --timeout 60s
      * 설명 : CPU 4코어 + 메모리 부하를 60초간 인위적으로 발생시킵니다(알람/스케일링 검증).
    * 7. 실시간 모니터링
      * 명령어 : htop
      * 설명 : CPU/메모리/프로세스를 실시간으로 관찰합니다. (종료: q)
  * 스토리지 I/O 성능 테스트 (fio)
    * 요약 : 마운트된 볼륨(Ceph/RBD 등)에서 FIO 로 랜덤/순차 읽기·쓰기 부하를 주고 IOPS·지연(Latency)·대역폭을 측정합니다.
    * 1. 도구 설치 (필요 시)
      * 명령어 : sudo apt install -y fio
      * 설명 : I/O 벤치마크 도구 fio 를 설치합니다.
      * 주의 : RHEL/CentOS 계열은 `sudo dnf install -y fio`
    * 2. 대상 볼륨으로 이동
      * 명령어 : cd /mnt/data
      * 설명 : 테스트할 마운트 볼륨 경로로 이동합니다.
      * 주의 : ⚠️ 운영 데이터가 있는 경로는 피하세요. 테스트 파일이 생성됩니다.
    * 3. 랜덤 쓰기 테스트
      * 명령어 : fio --name=randwrite --ioengine=libaio --iodepth=32 --rw=randwrite --bs=4k --direct=1 --size=1G --numjobs=1 --runtime=60 --group_reporting
      * 설명 : 4k 블록 랜덤 쓰기로 IOPS 와 지연(Latency)을 측정합니다.
    * 4. 순차 읽기 테스트
      * 명령어 : fio --name=seqread --ioengine=libaio --iodepth=32 --rw=read --bs=1m --direct=1 --size=1G --numjobs=1 --runtime=60 --group_reporting
      * 설명 : 1M 블록 순차 읽기로 처리량(대역폭)을 측정합니다.
    * 5. 테스트 파일 정리
      * 명령어 : rm -f randwrite.* seqread.*
      * 설명 : fio 가 생성한 테스트 파일을 삭제합니다.

    * 2. 강제 종료
      * 명령어 : sudo kill -9 <PID>
      * 설명 : 프로세스를 강제 종료해 장애 상황을 유발합니다.
      * 주의 : ⚠️ 운영 중인 서비스에는 사용하지 마세요. 테스트/검증 환경에서만 진행하세요.
    * 3. 자동 복구 확인
      * 명령어 : systemctl status <서비스명>
      * 설명 : systemd 의 Restart 설정에 따라 서비스가 자동 재시작됐는지 확인합니다.
    * 4. 서비스 로그 추적
      * 명령어 : journalctl -u <서비스명> -f
      * 설명 : 해당 서비스의 재시작/크래시 로그를 실시간으로 추적합니다. (종료: Ctrl+C)
    * 5. 시스템 로그 추적
      * 명령어 : tail -f /var/log/syslog
      * 설명 : 전체 시스템 로그를 실시간 추적합니다. (RHEL 계열은 /var/log/messages)
